Nine Websites

Inspiration

  1. 1. Joe's Cement Work
    I thought this website made good use of multimedia on the front page with being invasive. I also liked the review carousel which inspired me to create one for this website.
  2. 2. Toronto Concrete
    I liked how this website displayed their services on the home page, and created a tile grid similar to the one on this website, but utilized a background image and overlaid the text instead of having them separate.
  3. 3. Concrete Colour Solutions
    This website gave me the idea to create a colour grid to help potential customers easily visualize different concrete colours at their convenience.

Downloads

  1. 1. Feather Icons
    This site has almost 300 open source icons that can be downloaded as .svg, making it easy to scale up or down, change stroke width and fill colour.
  2. 2. Daisy UI
    A component library for Tailwind CSS, this website is very helpful for creating and managing colour themes and component templates using pure CSS.
  3. 3. Unsplash
    A really useful website to download high quality, royalty free images for free.

Technical

  1. 1. mdn
    Documentation for everything web development. Has great documentation and examples, and I rarely need to utilize another source.
  2. 2. Stack Overflow
    Really useful when trying to implement something complex. This website is helpful to understand others' experiences with a similar problem, and how the community helped them solve it. This helps me solve my own problems.
  3. 3. Tailwind CSS
    Documentation for CSS framework Tailwind CSS. I prefer Tailwind to regular CSS, and find it much more convienient when styling components.

Minimum Requirement Features

The URLs use a combination of links to pages on the website with anchor tags, and screenshots of code.

HTML5

  1. A. Appropriate HTML5 tags, including use of HTML5 structural tags:
    1. i. <main> - every page uses main after body
    2. ii. <section> Used sementically instead of divs for blocks of content
    3. iii. <aside> accompanied with sections for less important content
    4. iv. <nav> The navbar and in the footer
    5. v. <footer> The footer
    6. vi. <button> Used within many card components, form and navbar
    7. vi. <slot> Used to accept content in web components
    8. vii. <span> Mostly used to slot into web components
    9. viii. <p> Anywhere there's text
    10. ix. <video> Video hero
    11. x. <figure> Left/right card components
    12. xi. <ul> - Navbar
    13. xii. <ol> This page
    14. xiii. <table> <tr> <th> <td> Beside form
  2. B. Inline images both static and clickable - clickable images pop up as a modal so the user can see it larger
  3. C. Use of appropriate special characters - all of the tiles use a non-breaking space special character &nbsp;
  4. D. Use of appropriate meta tags - image of <meta> tags found in all page <head> sections
  5. E. A footer on the home page indicating when the website was last updated or modified - included on bottom right of footer

CSS3

  1. A. Use CSS3 to format the appearance of:
    1. paragraphs - The homepage uses the hero , left & right card , review card , and tile components which all contain paragraphs with different styling. Individual styles can be found in relevant component HTML.
    2. lists - there are many lists throughout the site, including the navbar, footer, and this page.
    3. headers - The homepage utilizes H1 for the Hero header text, H2 for subsequent sections, H3 for the Services tiles, and the footer uses H6 for its headers
    4. background images - Services tiles use the tile component, which uses a background image
    5. tables - simple table with zebra striping and scroll overflow
    6. forms - contact form with CSS and JS validation
  2. B. Use CSS3 contextual selectors (e1, e2, e3; e f; e > f; e + f) and at least two attribute selectors, excluding class and id attribute selectors.
  3. C. Use CSS3 styles for colour RGBA (colour tiles) & HSLA. (The grey square logo in header and footer)
  4. D. Use dynamic pseudo-classes to highlight hyperlinks states of link(visited, active, and hover), (see footer links) focus, (see form fields) and two structural pseudo-classes (first-of-type and last-of-type). (table on contact page for zebra striping)
  5. E. Use CSS3 background styles:
    1. i. background size, background-clip, background-origin Background services tile uses all three to ensure the image is contained within its width, clips to the padding, and it's origin is the content to give it the padded tile look
    2. ii. padding - lots of things use padding, like the form container, the buttons, the navbar
    3. iii. margins - all pages use a max-width and centering with margin 0 auto and h1's have top and bottom margin, including this page
    4. iv. borders and use of an appropriate border style are demonstrated on form inputs
    5. v. use of rounded corners - colours "tiles" is an example. Also all buttons have rounded corners
  6. F. Your home page should have a drop cap created with CSS3. Cards under the Hero on the homepage
  7. G. Define document content with the class and id properties, and create styles for them. Reviews carousel on homepage uses both class and id props for styles.
  8. H. Use each of the following page layout techniques:
    1. i. Grid Colour tiles are in a CSS grid
    2. ii. Flex Almost all components use Flex, navbar is good example of flex-row. Contact form is a good example of flex-col
    3. iii. Float Drop caps on home page and image on About us page use floats.
    4. iv. Positioning The review card Avatar image (the capital letter) uses Relative positioning, and the Google icon uses Absolute positioning to make it appear as a single icon
  9. I. Use CSS3 text and box shadows. All cards and tiles use box shadows. The drop caps use text shadow
  10. J. Use CSS3 transform rotate feature. The carousel on the homepage uses transform translateX to move the cards off and on the screen.
  11. K. The hero and the services tiles use opacity to create this effect

Web Form

  1. A. Use appropriate form elements including some of the new HTML5 form elements.Uses HTML5 input element types like email and tel to enforce regex patterns
  2. B. Apply appropriate styles to web forms. Form uses flex-col to organize inputs, labels and buttons.
  3. C. Apply inline form validation by using the focus pseudo-class to appropriate form elements. Valid, invalid, and focus pseudo-classes used to style borders around inputs (see link for all three examples respectively)

Multimedia

  1. A. Use sound and video somewhere on your website. Ensure these are of short duration, preferably about 30 seconds. (1st one is also video. 2nd may not autoplay depending on browser config)
  2. B. Use transitions (reviews carousel) and animations (logo in navbar and footer).

Javascript

  1. A. Use
    1. i. variables - variables used for function related to review carousel on homepage
    2. ii. arithmetic, comparison and logical operators updates the pixel width to translateX the review cards in the carousel. Uses arithmetic to calculate number of pixels, uses comparison and logical operators to check array length and perform actions depending on result
    3. iii. subroutines/functions - example of a function. Updated the background url for the tile web component
    4. iv. arrays an array of colour objects to programmatically generate colour tiles
    5. v. conditional statements; - a conditional to check if an elementID exists and executes navigateToTarget function if true
    6. vi. conditional operators; - checks BREAK constant, which is initial screen width and uses DESKTOP constant or MOBILE constant depending on the width. This will ensure three cards are left in view on desktop before resetting
    7. vii. running time commands with system clock - setInterval to move the review cards with the moveToNextItem function every 5 seconds
  2. B. Use system date (example: clock, countdown timer to an event) on your website. - Uses today's date and extracts the year to keep copyright up to date.
  3. C. Validate form input by using appropriate JavaScript functions. - uses JS to check validity (regex/required) and adds or removes styling on blur, so user's know their input is valid once leaving the input
  4. D. Use JavaScript alerts for validation purposes. - Uses an alert after the form submission to indicate if the form submitted successfully or not

Responsive Design

  1. A. Your website design should include responsive design implementation using concepts covered in this course that would accommodate desktop, tablet, and mobile designs, including
    1. i. positioning - mobile, tablet, desktop on desktop the image is larger and floated to the left, tablet and mobile use smaller image and block image
    2. ii. grid - mobile, tablet, desktop mobile uses 1 tile, tablet 2 tiles, and desktop 3 tiles per row
    3. iii. flex - mobile, tablet, desktop mobile uses smaller logo and hamburger menu, tablet uses larger logo, and desktop expands menu across navbar
  2. B. The use of contextual selectors to control sub-level menus is required, including a navicon image for mobile design. - menu icon appears on mobile and tablet, when clicked has dropdown of menu items
  3. C. Images should also be responsive for different screen sizes and devices.
    1. Mobile image
    2. Tablet image
    3. Desktop image
    images use width-fit and height-fit to fill it's container, then the container size is manipulated
